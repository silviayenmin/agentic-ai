# Project Manager (PM) Agent

## Role
The Project Manager agent takes the detailed plan from the Business Analyst, reviews it, and translates the sprints into a concrete, prioritized task backlog for the development team. It ensures the plan is ready for execution.

## Responsibilities
-   Review and validate the sprints generated by the BA.
-   Decompose sprints into individual development tasks.
-   Assign tasks (conceptually) to developers (though our current Dev agent is unified).
-   Prioritize tasks for efficient development flow.

## What it Does
-   Receives the `sprints` from the BA agent.
-   Uses an LLM (based on user's `provider` and `model` choice) to process the sprints.
-   Generates a structured list of tasks, each with a `title`, `description`, `assigned_to` (Developer), and `priority`.
-   Outputs this as a JSON list of tasks.

## Input Data
-   `state["sprints"]` (List[dict]): The list of sprints and their features, as defined by the BA agent.
-   `state["project_requirements"]` (string): (Context for the LLM) The overall project goal.
-   `state["provider"]` (string): The chosen LLM provider.
-   `state["model"]` (string): The specific LLM model ID.

## Output Data (Transfers to Dev Agent)
-   `state["tasks"]` (List[dict]): A list of dictionaries, each representing a task with details like `id`, `title`, `description`, `assigned_to`, and `priority`.
-   `state["logs"]` (List[str]): Appends log entries detailing its activity.
-   `state["agent_statuses"]["PM"]` (string): Updates its status to "working" then "done".
-   `state["current_agent"]` (string): Sets the current active agent to "PM".
